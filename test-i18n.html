Quick i18n Testing Methods

Method 1: Chrome DevTools (Recommended)

  1. Open Chrome DevTools (F12)
  2. Click the 3 dots menu → More tools → Sensors
  3. In the Sensors tab, find "Locale"
  4. Select "pt-BR" from dropdown
  5. Refresh the page

Method 2: Console Commands

Run these in browser console:

// Switch to Portuguese
localStorage.setItem('weddingAppLanguage', 'pt');
location.reload();

// Switch to English
localStorage.setItem('weddingAppLanguage', 'en');
location.reload();

// Clear preference (use browser detection)
localStorage.removeItem('weddingAppLanguage');
location.reload();

Method 3: Direct URL Testing

  • Portuguese: http://localhost:3000/?lang=pt
  • English: http://localhost:3000/?lang=en

Method 4: Browser Language Simulation

Use Chrome with command line flags:

# Mac/Linux
/Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome --lang=pt-BR

# Windows
chrome.exe --lang=pt-BR

Testing Checklist

  • ☐ Homepage loads in Portuguese for pt-BR browser
  • ☐ Language toggle switches all text immediately
  • ☐ Language preference persists after refresh
  • ☐ URL parameter overrides browser language
  • ☐ Forms and validation messages translate
  • ☐ Dynamic content (countdown, RSVP) translates
  • ☐ No missing translation keys (no raw keys shown)
